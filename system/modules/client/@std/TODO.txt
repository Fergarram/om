BLOB LOADER SUMMARY - NEXT STEPS FOR v1.0

==============================================================================

OVERVIEW
--------
The Blob Loader is a system for managing inline ES modules, CSS modules, and media 
files in HTML documents with support for remote sources, caching, and snapshots.

CURRENT STATE
-------------
Core functionality is implemented:
- Module/style/media loading with blob URLs
- Remote fetching with IndexedDB caching
- Snapshot system for saving/restoring document states
- Basic CRUD operations for modules

REMAINING TASKS FOR v1.0
------------------------

1. CRUD INTERFACE IMPROVEMENTS
   Current API exposes:
   - getCachedModule/setCachedModule/deleteCachedModule
   - getCachedMedia/setCachedMedia/deleteCachedMedia
   - runNonExportingModuleScript (stub)
   
   Need to implement:
   - Complete deleteCachedModule() and deleteCachedMedia() functions
   - Complete runNonExportingModuleScript() for dynamic module loading

2. CLONE DOCUMENT API (all stubs currently)
   Implement the following functions:
   - cloneDocument() - create in-memory copy of document
   - getCloneOuterHTML() - export clone as HTML string
   - downloadCloneAsHtmlFile() - download clone as HTML file
   - addCloneModule() - add new module to clone
   - getCloneModule() - retrieve module from clone
   - updateCloneModule() - modify existing module in clone
   - removeCloneModule() - delete module from clone
   - addCloneMediaModule() - add media to clone
   - getCloneMediaModule() - retrieve media from clone
   - updateCloneMediaModule() - modify media in clone
   - removeCloneMediaModule() - delete media from clone

3. SNAPSHOT FUNCTIONALITY
   Complete the stub:
   - saveSnapshotAsHtmlFile() - export snapshot as downloadable HTML

4. METADATA HANDLING
   - Store unrecognized HTML attributes as metadata on blob-module elements

5. NAMING CONSISTENCY (LAST)
   - Standardize terminology across codebase
   - Use: "media module", "script module", "style module"
   - All should be referred to as "blob modules of different kinds"
